## 五、（二）具体页面设计

**1、总览首页的设计**
　　总览首页是整个系统的入口，由顶部导航栏、轨道英雄区和专业示例画廊三部分组成。页面顶端为固定导航栏，包含“示例画廊”“网页拆解”与“开始生成”三个入口，点击可快速切换到对应功能。页面载入时先播放五等分切片转场的开场动画，随后展现中央主标题与环绕公转的专业卡片，并配合 WebGL 流体背景随鼠标搅动产生发光效果，营造电影级的视觉氛围。继续向下滚动即为 53 个专业的三维倾斜画廊卡片，鼠标悬停时卡片会轻微倾斜并浮现光晕，点击任意一张即可进入对应的示例页面。具体页面设计如图 2 所示。

**2、AI 生成工作台（Studio）的设计**
　　AI 生成工作台是系统的核心功能页，用户在此用一句话描述需求即可生成完整网页。页面左侧为指令输入区，用户输入所需的网页内容后点击生成，后端随即调用 DeepSeek 大模型实时流式返回单文件 HTML 代码。页面右侧为沙盒预览区，采用 iframe 隔离运行生成结果，做到即写即现。预览区还支持可视化结构编辑，用户可直接选中页面中的某一区块并输入修改要求，对其进行局部重写，而无需重新生成整页。此外，工作台还提供多页网站模式与历史生成记录查看功能。具体页面设计如图 3 所示。

## 六、核心问题的解决办法

**（一）提示词模板驱动的网页生成**
　　本系统最核心的问题在于：如何把用户输入的一句话需求，稳定地转化为一个完整、可直接运行且风格统一的网页。若将用户的原始描述直接提交给大模型，返回结果往往并不可控，可能夹带解释性文字、代码不完整、缺少交互或风格杂乱。为此，本设计采用“提示词模板加大模型生成”的方案：在用户需求之外再包裹一层系统提示词（System Prompt），用硬性规则约束模型只输出从 `<!DOCTYPE html>` 到 `</html>` 的完整单文件 HTML，统一使用 Tailwind 与原生 JavaScript，并强制页面包含滚动渐入、响应式导航、轮播等真实交互。整体流程为：前端先将用户需求通过 `/api/generate` 接口提交到后端，后端调用 `buildSystemPrompt` 把系统提示词与用户需求拼装成 system、user 两条消息，再请求 DeepSeek 的 `/chat/completions` 接口；模型返回 HTML 源码后，后端剥除可能存在的 Markdown 代码围栏并截取出完整文档，最后存入数据库并回传前端，在沙盒中即时渲染。生成流程如图 6 所示。

图 6　提示词到页面的生成流程

```js
messages: [
  { role: "system", content: buildSystemPrompt(options) }, /*提示词模板：约束输出规则*/
  { role: "user",   content: prompt },                     /*用户的一句话需求*/
]
```

**（二）自动轮播图的实现**
　　为了让校友评价、作品图片等内容能够循环展示，需要实现一个稳定的自动轮播。本设计将各张幻灯片以绝对定位叠放在同一容器内，通过切换透明度 `opacity` 实现淡入淡出，而非使用容易算错位置的横向位移。轮播由一个 `show(i)` 函数统一控制，每次只让当前一张显示、其余隐藏，索引统一采用取模运算以保证首尾循环。同时通过定时器每隔一段时间自动切换，并在鼠标移入时暂停、移出时恢复，再配合左右箭头与底部圆点供用户手动切换。其中定时器的间隔即可控制轮播速度，当前设置为 5000 毫秒。轮播效果如图 7 所示。

图 7　校友评价轮播页面

```js
timer = setInterval(next, 5000); /*每 5000 毫秒自动切换一张，数值越小切换越快*/
```

**（三）报名表单的实时校验**
　　在线报名表单需要在用户提交之前就发现填写错误，以提升填写体验。本设计为手机号输入框绑定了 `input` 事件，用户每输入一个字符都会实时进行格式校验：先过滤掉非数字字符并限制为 11 位，再用正则表达式判断是否为合法手机号，并即时给出“格式正确”或“格式不正确”的彩色提示。只有当姓名非空且手机号通过校验时，提交按钮才会真正记录这条预约，否则会自动定位到对应输入框并给出提示。校验效果如图 8 所示。

图 8　报名表单实时校验页面

```js
var RE = /^1[3-9]\d{9}$/;          /*中国大陆手机号校验规则*/
if (RE.test(phone.value)) { ... }  /*格式正确时给出绿色提示，否则提示错误*/
```

## 七、小结和展望

　　在完成这次毕业设计的过程中，我独立搭建了一个“对话即渲染”的 AI 网页生成系统，学会了如何用 Node.js 与 Express 搭建轻量后端代理，如何调用 DeepSeek 大模型，并把前端、后端与人工智能接口串联成一条完整的生成链路。其中我体会最深的是“提示词工程”的重要性：同样的大模型，直接把用户需求丢进去结果杂乱无章，而在外层包裹一套约束严格的系统提示词模板后，模型就能稳定地输出风格统一、交互齐全的完整网页。开发中我也遇到并解决了不少具体问题，例如用透明度切换实现稳定的自动轮播图并控制其播放速度、用正则表达式为表单做实时校验、用 iframe 沙盒隔离运行生成结果等。解决这些问题不仅锻炼了我的编码能力，也让我真正理解了一个交互系统从需求到落地是如何运转的。通过这个系统，我希望让不懂代码的人也能用一句话快速生成一个可用的网页，把繁琐的开发变得简单。

　　展望未来，这个系统还有很大的改进空间。在功能上，我计划支持接入更多大模型、提供更精细的可视化局部编辑、扩充更多设计风格模板，并进一步优化移动端的适配与生成速度。这次实践教会我的，不仅是某一门具体技术的用法，更重要的是面对陌生领域时如何快速查阅资料、动手试错并最终把它拿下的能力。我相信，在不断变化的计算机与人工智能领域，保持好奇、持续学习并勇于尝试，永远是最值得坚持的事。
